# AI 提示词记录

## 原始需求

根据 `评判标准.md` 做一个程序，能够取跟一个指定汉字组成昵称的程序，可以指定生成的昵称是几个字的，并且要求全是汉字，需要全部列举出来并打分评级，区分筛选，支持人工选择喜欢的一批成组，可以多次筛选，支持排除后不再生成，将生成的昵称存放本地。

## 实现记录

- 使用 Vite + React 创建本地 Web 应用。
- 根据 `评判标准.md` 将 100 分评分体系拆分为可计算的启发式规则。
- 实现指定汉字、指定字数、风格、汉字位置、生成数量等输入项。
- 实现纯汉字校验，非汉字昵称不会进入候选列表。
- 实现 S/A/B/C/D 评级筛选和关键词筛选。
- 实现“喜欢”列表，用于人工选择喜欢的一批昵称。
- 实现“排除”列表，被排除昵称后续不再生成。
- 使用浏览器 `localStorage` 保存喜欢列表、排除列表和生成记录。
- 实现 JSON 导出，便于将筛选结果保存到本地文件。

## 关键约束

- 页面消息、日志、文档均使用中文。
- 不使用数据库，因此无需提供初始化 SQL。
- 程序需能通过 `npm run dev` 本地启动。

## AI 使用率查询记录

## AI 使用率查询实现记录

- 更新 AI 提示词，要求 AI 为每个昵称返回 `usageRate`、`usageLevel`、`usageSource`。
- AI 使用率基于模型通用语料认知、常见昵称构成和公开网络常见度经验估算。
- 解析 AI 结果时优先使用 AI 返回的使用率。
- 如果 AI 未返回可用使用率，则自动回退到本地规则估算。
- 卡片继续通过原有使用率标签和来源说明展示结果。
