# 汉字昵称生成评选器 — 远程盒子部署指南

适用于 PocketClaw 盒子 (Linux aarch64)

---

## 环境信息

| 项目 | 值 |
|------|-----|
| 盒子 IP | `192.168.0.106` |
| 普通用户 | `pocketclaw` |
| 源码路径 | `/root/.pocketclaw/workspaces/assistant/data/self-name` |
| 静态文件路径 | `/var/www/self-name` |
| 本地路径 | `C:\Users\muxiao\Desktop\windsurf-project\self-name` |
| 项目类型 | Vite + React 静态 SPA |
| 访问端口 | `8080`（80 被 PocketClaw 占用） |

## 方案一：本地构建 + 上传 dist（推荐）

适合盒子上没有 Node.js 环境的情况，只需 nginx 即可运行。

### 步骤 1: 本地构建

在本地 PowerShell 执行：

```powershell
cd C:\Users\muxiao\Desktop\windsurf-project\self-name
npm run build
```

构建产物在 `dist/` 目录。

### 步骤 2: 登录盒子创建目录

```powershell
ssh pocketclaw@192.168.0.106
```

```bash
sudo -i
mkdir -p /var/www/self-name
```

### 步骤 3: 上传 dist 文件

在本地 PowerShell 执行：

```powershell
scp -r C:\Users\muxiao\Desktop\windsurf-project\self-name\dist\* root@192.168.0.106:/var/www/self-name/
```

### 步骤 4: 配置 nginx

```bash
ssh pocketclaw@192.168.0.106
sudo -i
```

#### 4a. 安装 nginx（如果没有）

```bash
# Ubuntu/Debian
apt-get update && apt-get install -y nginx

# CentOS/RHEL
yum install -y nginx

# Alpine
apk add nginx
```

#### 4b. 禁用默认站点（端口 80 被占用时）

```bash
rm -f /etc/nginx/sites-enabled/default
```

#### 4c. 创建站点配置

```bash
cat > /etc/nginx/conf.d/self-name.conf << 'EOF'
server {
    listen 8080;
    server_name _;

    root /var/www/self-name;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|svg|ico|webp|woff|woff2)$ {
        expires 30d;
        add_header Cache-Control "public, max-age=2592000, immutable";
        try_files $uri =404;
    }

    gzip on;
    gzip_types text/plain text/css application/json application/javascript image/svg+xml;
}
EOF
```

#### 4d. 修复权限并启动 nginx

```bash
chown -R www-data:www-data /var/www/self-name
nginx -t
nginx -s stop 2>/dev/null; nginx
```

### 步骤 5: 验证

在盒子上测试：

```bash
curl -s http://localhost:8080 | head -5
```

应看到 HTML 输出：

```html
<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
```

在浏览器访问：`http://192.168.0.106:8080`

> **注意：** 本地构建时 `vite.config.js` 的 `base` 需要改为 `'/'`（盒子直接用根路径访问），构建完成后再改回 `'/self-name/'`（腾讯云部署用）。

## 方案二：盒子上构建运行

适合盒子上有 Node.js 环境，需要修改代码后热更新的场景。

### 步骤 1: 上传源码

在本地 PowerShell 执行：

```powershell
scp -r C:\Users\muxiao\Desktop\windsurf-project\self-name root@192.168.0.106:/root/.pocketclaw/workspaces/assistant/data/
```

### 步骤 2: 安装依赖并构建

```bash
ssh pocketclaw@192.168.0.106
sudo -i

cd /var/www/self-name

# 安装 Node.js（如果没有）
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs

# 安装依赖
npm install

# 构建
npm run build
```

### 步骤 3: 配置 nginx

同方案一的步骤 4。

### 步骤 4: 验证

```bash
curl -s http://localhost:8080 | head -5
```

浏览器访问：`http://192.168.0.106:8080`

## 方案三：Docker 部署

适合盒子上有 Docker 的场景，一键运行。

### 步骤 1: 上传整个项目

```powershell
scp -r C:\Users\muxiao\Desktop\windsurf-project\self-name root@192.168.0.106:/root/.pocketclaw/workspaces/assistant/data/
```

### 步骤 2: 构建并运行 Docker 镜像

```bash
ssh pocketclaw@192.168.0.106
sudo -i

cd /var/www/self-name

# 构建镜像
docker build -t self-name .

# 运行容器
docker run -d \
  --name self-name \
  --restart unless-stopped \
  -p 8080:80 \
  self-name
```

### 步骤 3: 验证

```bash
docker ps | grep self-name
curl -s http://localhost:8080 | head -5
```

浏览器访问：`http://192.168.0.106:8080`

## 故障排查

### 问题 1: SCP 上传失败

```
scp: lost connection
```

**解决：**
- 检查网络连接
- 确认密码正确
- 尝试用 SFTP：
  ```bash
  sftp root@192.168.0.106
  sftp> put -r C:\Users\muxiao\Desktop\windsurf-project\self-name\dist /var/www/self-name
  ```

### 问题 2: 权限被拒绝

```
Permission denied
```

**解决：**
```bash
sudo -i
chmod -R 755 /var/www/self-name
chown -R root:root /var/www/self-name
```

### 问题 3: nginx 端口冲突

```
nginx: [emerg] bind() to 0.0.0.0:8080 failed (98: Address already in use)
```

**解决：**
```bash
# 查看占用端口的进程
ss -tlnp | grep 8080

# 换一个端口，编辑配置
vi /etc/nginx/conf.d/self-name.conf
# 把 listen 8080 改成其他端口，如 8081

nginx -t && nginx -s reload
```

### 问题 4: 页面空白 / 404

**解决：**
```bash
# 检查文件是否存在
ls -la /var/www/self-name/index.html

# 检查 nginx 配置中的 root 路径是否正确
cat /etc/nginx/conf.d/self-name.conf

# 检查 nginx 错误日志
tail -20 /var/log/nginx/error.log
```

### 问题 5: 静态资源加载失败（JS/CSS 404）

**解决：**
```bash
# 检查 assets 目录
ls -la /var/www/self-name/assets/

# 确认 vite.config.js 中 base 路径正确
cat /var/www/self-name/vite.config.js
# 应该是 base: '/self-name/' 或 base: '/'（取决于部署方式）
```

> **注意：** 如果直接通过 `http://IP:端口/` 访问（无子路径），需要将 `vite.config.js` 中的 `base` 改为 `'/'`，然后重新构建。

### 问题 6: Docker 构建失败

**解决：**
```bash
# 检查 Docker 是否运行
docker info

# 查看构建日志
docker build -t self-name . 2>&1

# 如果是网络问题，使用国内镜像
docker build --network=host -t self-name .
```
