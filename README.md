# 汉字昵称生成评选器

一个本地运行的汉字昵称生成与筛选程序。用户可以指定一个核心汉字，设置昵称字数，批量生成纯汉字昵称，并根据 `评判标准.md` 自动打分评级。

## 功能说明

- **指定汉字生成**：输入一个汉字，生成所有候选昵称都会包含该汉字。
- **指定昵称字数**：支持 2-6 字昵称生成。
- **全量汉字库**：使用 Unicode CJK 基本汉字区作为生成字库，并优先排列风格汉字。
- **纯汉字约束**：程序只保留全汉字昵称，不生成字母、数字或符号昵称。
- **评分评级**：按读音韵律、视觉排版、意境风格、场景适配四类规则计算 100 分，并给出 S/A/B/C/D 评级。
- **使用率估算**：基于常见汉字、风格字频、昵称长度和重复结构估算使用率，并在卡片中标明数据来源。
- **账号名称合规过滤**：生成时过滤低俗、违规、仿冒政党政府机关、新闻媒体、国家或国际组织等风险内容。
- **AI 生成**：支持接入 OpenAI 兼容接口，根据 `评判标准.md` 生成并返回带评分的候选昵称。
- **区分筛选**：支持按评级和关键词筛选候选昵称。
- **人工成组选择**：点击“喜欢”可将昵称加入喜欢列表，形成喜欢的一批。
- **切换列表查看**：支持在候选、喜欢、排除三个列表之间切换查看。
- **撤销操作**：支持取消喜欢，也支持取消排除，让昵称重新参与后续生成。
- **多次筛选**：可反复调整汉字、字数、风格、位置和数量，多轮生成筛选。
- **排除不再生成**：点击“排除”后，该昵称会进入排除列表，后续生成自动跳过。
- **本地保存**：喜欢列表、排除列表、生成记录全部保存在浏览器 `localStorage` 中。
- **导出结果**：支持将本地筛选结果导出为 JSON 文件。

## 使用率数据来源

当前版本使用本地规则估算，不调用外部接口：

- 常见汉字命中数量
- 当前风格字命中数量
- 昵称长度
- 重复结构

该使用率不代表全网真实注册数量。后续如需真实统计，可接入搜索引擎公开结果数、公开平台搜索结果或自建昵称样本库。

## AI 生成功能

页面提供 AI 生成配置：

- API 地址：OpenAI 兼容的 `chat/completions` 接口地址
- 模型名：例如 `gpt-4o-mini`
- API Key：仅保存到当前浏览器 `localStorage`

内置服务预设：

- DeepSeek：`https://api.deepseek.com/chat/completions`，默认模型 `deepseek-chat`
- OpenAI：`https://api.openai.com/v1/chat/completions`，默认模型 `gpt-4o-mini`

AI 生成会把 `评判标准.md` 的核心规则写入提示词，要求返回 JSON 数组。AI 结果会优先使用 AI 返回的使用率估算；如果 AI 未返回使用率，则回退到本地规则估算。当前为前端直连方式，生产环境更推荐使用后端代理保存 API Key，避免密钥暴露。

## 开源注意事项

项目已提供 `.gitignore`，会忽略依赖目录、构建产物、环境变量、本地导出结果和可能包含密钥的私有数据文件。

开源前请确认：

- 不要提交真实 API Key。
- 不要提交 `.env`、本地导出的筛选结果或个人测试数据。
- 如果需要提供环境变量示例，请使用 `.env.example`。

## 项目启动方式

首次运行请安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

打包构建：

```bash
npm run build
```

预览生产构建：

```bash
npm run preview
```

## 运行端口

- 开发服务端口：`5173`
- 生产预览端口：`4173`
- Docker / Nginx 部署端口：`80`

## 服务器部署方式

### 方式一：普通静态部署

在本地或服务器执行构建：

```bash
npm install
npm run build
```

构建完成后，将 `dist` 目录上传到服务器的网站根目录即可，例如 Nginx 的静态目录。

推荐 Nginx 配置可参考项目根目录：

```text
nginx.conf
```

### 方式二：Docker 部署

项目已提供 `Dockerfile` 和 `.dockerignore`。

构建镜像：

```bash
docker build -t hanzi-nickname .
```

启动容器：

```bash
docker run -d --name hanzi-nickname -p 80:80 hanzi-nickname
```

访问地址：

```text
http://服务器IP/
```

### 方式三：已有 Nginx 服务器部署

将 `npm run build` 生成的 `dist` 目录内容复制到服务器静态目录，例如：

```text
/usr/share/nginx/html
```

如果部署在子路径，请同步调整服务器转发规则；默认推荐部署在域名根路径。

## 数据存储说明

本项目不使用数据库，所有筛选数据存储在浏览器本地 `localStorage` 中。由于没有数据库组件，因此无需初始化 SQL 语句。

本地存储键名：

```text
hanzi-nickname-studio-v1
```

导出的文件名：

```text
昵称筛选结果.json
```

## 评分规则来源

评分规则参考根目录 `评判标准.md`：

- 读音韵律：30 分
- 视觉排版：25 分
- 意境风格：30 分
- 场景适配与实用性：15 分
- 评级：S/A/B/C/D

程序会将主观规则转化为可计算的启发式评分，方便批量初筛；最终仍建议人工选择最喜欢的一批。
